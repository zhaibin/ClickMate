## 🪟 Windows 打包

### 方式1：便携版 ✨推荐

**命令：**
```bash
scripts\build_release.bat
```

**自动完成：**
1. 检查 DLL 文件
2. 构建 Release 版本
3. 创建便携版文件夹
4. 打包 ZIP 压缩包

**输出结果：**
```
releases\v2.2.2\
├── ClickMate_v2.2.2_Portable\
│   ├── clickmate.exe
│   ├── flutter_windows.dll
│   ├── mouse_controller.dll
│   ├── data\
│   └── (其他插件 DLL)
└── ClickMate_v2.2.2_Portable.zip
```

> 实际版本号由根目录 `VERSION` 文件读取，当前版本为 `2.2.2`。上面的目录应随 `VERSION` 自动变化为 `releases\v版本号\`。

### 方式2：单 EXE 打包

**需要工具：** Enigma Virtual Box（免费）
- 下载：https://enigmaprotector.com/en/downloads.html

**步骤：**
1. 先运行 `build_release.bat` 生成便携版
2. 使用 Enigma Virtual Box 打包所有文件为单个 EXE

### 方式3：安装程序

**需要工具：** Inno Setup（免费）
- 下载：https://jrsoftware.org/isdl.php

**步骤：**
1. 先运行 `build_release.bat` 生成便携版
2. 创建 Inno Setup 脚本打包为安装程序

### 手动打包

```cmd
# 1. 构建 Release 版本
flutter build windows --release

# 2. 复制 DLL
copy native\src\mouse_controller.dll build\windows\x64\runner\Release\

# 3. 打包整个 Release 文件夹
```

## 🍎 macOS 打包

### ⚠️ 重要说明：Native Library 打包

macOS 应用需要将 `libmouse_controller.dylib` 正确打包到应用包内。

**问题：** 如果只运行 `flutter build macos --release`，dylib 不会自动包含在 `.app` 包内，导致分发给用户后应用无法启动，报错：`Unsupported operation: Could not load libmouse_controller.dylib`

**解决方案：** 使用 `scripts/build_dmg.sh` 脚本，它会自动：
1. 编译 native library（如果需要）
2. 构建 Flutter 应用
3. 将 dylib 复制到 `ClickMate.app/Contents/Frameworks/`
4. 设置正确的 install_name（使用 `@executable_path/../Frameworks/`）
5. 创建 DMG 安装包

### 方式1：DMG 安装包 ✨推荐

**命令：**
```bash
bash scripts/build_dmg.sh
```

**输出结果：**
```
releases/v2.2.2/
├── ClickMate_v2.2.2_macOS.dmg
└── (构建的 .app 在 build/macos/Build/Products/Release/)
```

> 实际版本号由根目录 `VERSION` 文件读取，当前版本为 `2.2.2`。脚本会输出到 `releases/v版本号/ClickMate_v版本号_macOS.dmg`。

**自动完成：**
1. 清理旧构建
2. 获取依赖
3. 编译 native library（libmouse_controller.dylib）
4. 构建 Release 版本
5. **将 dylib 复制到 app bundle 的 Frameworks 目录**
6. **设置正确的 dylib install_name**
7. 创建 DMG 安装包

**分发方式：**
- 直接分发 `.dmg` 文件
- 用户打开 DMG，拖动应用到「应用程序」文件夹

### 方式2：App Bundle

**命令：**
```bash
flutter build macos --release
```

**输出结果：**
```
build/macos/Build/Products/Release/ClickMate.app
```

**⚠️ 重要：** 手动打包时必须将 dylib 复制到应用包内：
```bash
# 复制 dylib 到应用包
FRAMEWORKS_DIR="build/macos/Build/Products/Release/ClickMate.app/Contents/Frameworks"
mkdir -p "$FRAMEWORKS_DIR"
cp native/src/libmouse_controller.dylib "$FRAMEWORKS_DIR/"

# 设置正确的 install_name
install_name_tool -id "@executable_path/../Frameworks/libmouse_controller.dylib" \
    "$FRAMEWORKS_DIR/libmouse_controller.dylib"
```

**分发方式：**
1. 将 `.app` 文件压缩为 `.zip`
2. 用户解压后拖入「应用程序」文件夹

### macOS 代码签名（正式分发推荐）

**需要：**
- Apple Developer 账号
- 代码签名证书

```bash
# 签名应用
codesign --force --deep --sign "Developer ID Application: Your Name" \
  build/macos/Build/Products/Release/ClickMate.app

# 签名 DMG
codesign --force --sign "Developer ID Application: Your Name" \
  releases/v2.2.2/ClickMate_v2.2.2_macOS.dmg

# Notarization（公证，可选但推荐）
xcrun notarytool submit ClickMate_v2.2.2_macOS.dmg \
  --apple-id YOUR_ID --password YOUR_PASSWORD --team-id YOUR_TEAM
```

## 🔧 平台特定注意事项

### Windows

1. **DLL 依赖**
   - `flutter_windows.dll` - Flutter 运行时
   - `mouse_controller.dll` - 鼠标控制库
   - 插件 DLL（screen_retriever, window_manager 等）

2. **运行时要求**
   - Visual C++ Redistributable 2022
   - Windows 10 1809+

3. **快捷键注册**
   - 默认快捷键为 `Ctrl+Shift+1/2`
   - 如注册失败，通常是快捷键冲突、权限限制或 DLL 未加载
   - 建议在发布说明中提示用户：失败时可尝试管理员身份运行

### macOS

1. **动态库**
   - `libmouse_controller.dylib` 会被自动打包到 App Bundle

2. **权限要求**
   - 辅助功能权限（Accessibility）
   - 首次运行会提示用户授权

3. **代码签名**
   - 未签名的 App 会显示「无法验证开发者」
   - 用户需要在「系统偏好设置 → 安全性与隐私」中允许运行

4. **Gatekeeper**
   - 建议进行代码签名和公证
   - 否则用户需要右键点击 App 选择「打开」

## ⚠️ 常见问题

### Q: Windows 打包后无法运行？
**A:** 检查是否缺少 DLL 文件，特别是 `mouse_controller.dll`

### Q: macOS App 提示「已损坏」？
**A:** 这是 Gatekeeper 限制，运行以下命令：
```bash
xattr -cr /path/to/ClickMate.app
```

### Q: macOS 快捷键无响应？
**A:** 检查辅助功能权限是否已授予

### Q: 文件太大怎么办？
**A:** Flutter 最小运行时约 15-20MB，无法进一步压缩

### Q: 如何减少打包体积？
- 移除未使用的资源
- 使用 `flutter build --release`
- 考虑使用压缩工具
