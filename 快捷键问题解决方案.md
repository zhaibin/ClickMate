# 快捷键问题诊断和解决方案

## 🔍 问题分析

### 诊断结果
通过 `diagnose.bat` 诊断工具发现：

✅ **DLL文件正常** - `mouse_controller.dll` (132KB) 存在  
✅ **Flutter环境正常** - Flutter 3.38.1  
✅ **系统兼容** - Windows 10.0.26100  
❌ **权限不足** - 当前非管理员权限 ← **主要问题**

### 根本原因

**Windows全局热键注册限制**：
- Windows API `RegisterHotKey()` 用于注册系统级全局热键
- 在某些Windows版本和配置下，注册全局热键需要管理员权限
- 没有管理员权限时，`RegisterHotKey()` 会返回失败
- 这导致快捷键无法注册和使用

## ✅ 解决方案

### 方案1：以管理员身份运行（推荐）⭐

#### 步骤1：使用管理员启动脚本
```batch
右键点击 run_as_admin.bat → 选择"以管理员身份运行"
```

#### 步骤2：观察控制台日志
启动后查看控制台输出：
```
========================================
鼠标控制服务初始化中...
========================================
✓ 热键系统初始化: 成功
✓ 默认热键注册 Ctrl+Shift+S (VK:0x53): 成功
✓ 热键监听已启动
========================================
```

如果看到"成功"，表示快捷键已正常工作！

### 方案2：修改应用启动方式

#### 设置应用始终以管理员身份运行：
1. 右键点击 `run_debug.bat`
2. 选择"属性"
3. 点击"快捷方式"选项卡
4. 点击"高级"按钮
5. 勾选"以管理员身份运行"
6. 点击"确定"

### 方案3：使用其他快捷键组合（不推荐）

某些快捷键组合可能不需要管理员权限：
- 尝试F1-F12功能键
- 尝试不常用的字母键
- 但这不保证一定成功

## 🛠️ 故障排除

### 问题1：仍然无法注册快捷键（即使有管理员权限）

**可能原因**：快捷键被其他程序占用

**解决方法**：
1. 点击应用右上角⌨️图标
2. 更换其他不常用的快捷键
3. 推荐尝试：K、P、F9、F10等

**观察日志**：
```
========================================
更换快捷键...
✓ 旧热键已注销
✓ 新热键注册 Ctrl+Shift+K (VK:0x4B): 成功
启动热键监听定时器（每100ms检查一次）...
========================================
```

### 问题2：DLL加载失败

**症状**：
```
× 热键系统初始化失败！
  原因: DLL加载失败或Windows API调用失败
```

**解决方法**：
1. 运行 `diagnose.bat` 确认DLL存在
2. 重新复制DLL：`Copy-Item "native\src\mouse_controller.dll" "." -Force`
3. 确认DLL未被占用（重启应用）
